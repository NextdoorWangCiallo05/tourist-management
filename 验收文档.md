## 三、核心业务测试场景

### 场景 1：完整的报名 → 付费 → 完成流程

```
1. 登录系统                                          → 跳转仪表盘      □
2. 进入"旅游团查询" → 选择一个有余位的旅游团              → 看到剩余名额   □
3. 点击"立即报名"                                     → 跳转新建申请   □
4. 填写责任人信息、人数 → 确认订金金额 → 提交申请          → 生成申请编号   □
5. 在详情页添加参加者（姓名+证件号码）                    → 参加者列表更新  □
6. 支付订金                                           → 订金已支付     □
7. 完成申请                                           → 状态变为已完成  □
8. 进入"确认书打印" → 下载 PDF                         → PDF 含中文内容  □
9. 进入"财务导出" → 导出 Excel                         → 含该笔支付记录  □
10. 进入"操作日志" → 查看记录                           → 看到全流程日志  □
```

### 场景 2：取消申请 → 退款计算

```
1. 找一个待处理的申请                                   → 显示"处理中"   □
2. 点击"取消" → 确认                                    → 提示退款金额   □
3. 查看申请列表                                         → 状态变为已取消 □
4. 查看操作日志                                         → 有取消记录     □
```

### 场景 3：路线 → 旅游团 → 公开 → 报名

```
1. 新增一条路线                                         → 创建成功      □
2. 在该路线下新增旅游团（设置价格/容量/日期）               → 创建成功      □
3. 公开该旅游团                                         → 标签变绿      □
4. 进入旅游团查询页 → 该旅游团可见                        → 可看到        □
5. 停用该路线 → 旅游团管理页该路线旅游团仍可管理             → 不影响已有数据 □
```
