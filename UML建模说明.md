# 智游云管 - UML 建模说明

> 项目：旅游业务管理系统  
> 技术栈：Vue 3 + Flask + SQLite

---

## 一、用例图 (Use Case Diagram)

### 1.1 参与者 (Actors)

| 参与者 | 说明 |
|--------|------|
| **操作员** | 系统的主要使用者，负责日常业务操作 |
| **管理员** | 拥有全部权限，可管理旅游团、路线等基础数据 |

### 1.2 用例列表

#### 模块1：认证管理
| 编号 | 用例名称 | 参与者 | 说明 |
|------|---------|--------|------|
| UC-01 | 登录系统 | 操作员 | 输入用户名密码，获取 JWT Token |
| UC-02 | 退出登录 | 操作员 | 清除 Token，返回登录页 |

#### 模块2：首页概览
| 编号 | 用例名称 | 参与者 | 说明 |
|------|---------|--------|------|
| UC-03 | 查看首页统计 | 操作员 | 查看今日申请数、参加者数、收款金额、进行中团数 |
| UC-04 | 查看待办提醒 | 操作员 | 查看即将出发、待确认余款等提醒 |

#### 模块3：路线管理
| 编号 | 用例名称 | 参与者 | 说明 |
|------|---------|--------|------|
| UC-05 | 新增路线 | 管理员 | 创建新旅游路线（代码、名称、描述） |
| UC-06 | 编辑路线 | 管理员 | 修改路线信息，自动记录版本历史 |
| UC-07 | 停用路线 | 管理员 | 将路线标记为停用状态 |
| UC-08 | 查看路线列表 | 操作员 | 查看所有路线及状态 |

#### 模块4：旅游团管理
| 编号 | 用例名称 | 参与者 | 说明 |
|------|---------|--------|------|
| UC-09 | 新增旅游团 | 管理员 | 创建旅游团（关联路线、日期、价格、容量） |
| UC-10 | 编辑旅游团 | 管理员 | 修改旅游团基本信息 |
| UC-11 | 修改价格 | 管理员 | 修改成人/儿童价格，自动记录价格历史 |
| UC-12 | 公开旅游团 | 管理员 | 将旅游团设为公开，供客户报名 |
| UC-13 | 取消公开 | 管理员 | 将旅游团设为不公开 |
| UC-14 | 查询旅游团 | 操作员 | 按名称/代码搜索已公开旅游团 |
| UC-15 | 检查可用性 | 操作员 | 检查旅游团是否还有剩余名额 |

#### 模块5：申请管理
| 编号 | 用例名称 | 参与者 | 说明 |
|------|---------|--------|------|
| UC-16 | 新建申请 | 操作员 | 选择旅游团、填写责任人信息 |
| UC-17 | 添加参加者 | 操作员 | 录入参加者姓名、证件信息 |
| UC-18 | 支付订金 | 操作员 | 支付订金（按距出发天数计算比例） |
| UC-19 | 支付余款 | 操作员 | 支付剩余款项 |
| UC-20 | 完成申请 | 操作员 | 确认参加者信息完整后完成申请 |
| UC-21 | 取消申请 | 操作员 | 取消申请，计算退款金额 |
| UC-22 | 查看申请列表 | 操作员 | 按条件搜索和查看申请 |
| UC-23 | 查看申请详情 | 操作员 | 查看申请完整信息 |

#### 模块6：财务与打印
| 编号 | 用例名称 | 参与者 | 说明 |
|------|---------|--------|------|
| UC-24 | 导出财务数据 | 操作员 | 导出昨日收款数据 |
| UC-25 | 下载CSV | 操作员 | 将财务数据导出为 CSV 文件 |
| UC-26 | 打印确认书 | 操作员 | 打印昨日完成的申请确认书 |

### 1.3 用例关系

```
UC-16 新建申请 --extends--> UC-17 添加参加者
UC-16 新建申请 --extends--> UC-18 支付订金
UC-18 支付订金 --extends--> UC-20 完成申请
UC-20 完成申请 --extends--> UC-19 支付余款
UC-09 新增旅游团 --include--> UC-05 新增路线（需先有路线）
UC-12 公开旅游团 --include--> UC-09 新增旅游团（需先创建）
```

### 2.2 类之间的关系

| 关系 | 源类 | 目标类 | 多重性 | 说明 |
|------|------|--------|--------|------|
| 组合 | Route | TourGroup | 1 → * | 一条路线可以有多个旅游团 |
| 组合 | TourGroup | Application | 1 → * | 一个旅游团可以有多个申请 |
| 组合 | Application | Participant | 1 → * | 一个申请可以有多个参加者 |
| 组合 | Application | PaymentRecord | 1 → * | 一个申请可以有多个支付记录 |
| 组合 | TourGroup | PriceHistory | 1 → * | 一个旅游团可以有多个价格历史 |
| 组合 | Route | RouteHistory | 1 → * | 一条路线可以有多个版本历史 |
